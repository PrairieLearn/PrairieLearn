You can use the element pl-file-upload when you want to manually grade answers from students, or if you just want to have records of their solution process. By default, only file names listed in file-names can be uploaded and all files in the list must be provided.

Besides mandatory file names, pl-file-upload supports 3 other attributes to customize which files students can upload in a more fine-grained way:

  • optional-file-names defines list of file names that students can (are not required to) upload.
  • file-patterns defines list of file names that can include wildcard patterns and therefore allow a range of different names. Exactly one file must be provided to match each pattern.
  • optional-file-patterns defines another list of wildcard patterns that can include wildcards. For each pattern, any number of files may be uploaded.
A single pl-file-upload element can use any combination of mandatory and optional file names and patterns. Patterns can be repeated as shown above, but we strongly discourage the use of overlapping patterns in the same attribute. Please refer to the documentation for a more detailed explanation on how the different file lists interact.

Note that we use multiple pl-file-upload elements here for illustrative purposes, but that using more than one for real questions is not recommended.
