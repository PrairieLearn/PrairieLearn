Questions below were designed to showcase the different features of pl_number_input. The goal of pl_number_input is to allow users to enter real or imaginary values to be checked within a specific tolerance.

Within this question, you can observe the default behavior of the pl_number_input element. In particular, the default input field has help text as a placeholder as well as a question button and is set to use "relabs" to check the tolerance, which is a combination of relative ("rtol") and absolute ("atol") tolerances.

Under this variant, we opt to change the default comparison of pl_number_input from relative and absolute tolerances (comparison="relabs") to significant digits (comparison="sigfigs"), which has a default of two digits. In addition, we've opted to add a suffix or a unit classification to the input via suffix="$units$".
